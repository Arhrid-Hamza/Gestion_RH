Le projet a été conçu en adoptant une architecture client-serveur classique, où le frontend est développé avec React pour offrir une interface utilisateur dynamique et réactive, tandis que le backend utilise Node.js avec MongoDB pour gérer la logique métier, les données et les API. Cette séparation permet une meilleure modularité et facilite la maintenance.

Pour la gestion de l'état global de l'application, Redux a été choisi. Cette bibliothèque permet de centraliser l'état et de faciliter la communication entre les différents composants, assurant ainsi une cohérence des données affichées.

La base de données retenue est NoSQL, MongoDB, afin de garantir une flexibilité dans la gestion des données non structurées et une scalabilité horizontale efficace.

La sécurité est un aspect primordial du projet. Un système d'authentification et d'autorisation basé sur les rôles a été mis en place pour protéger les données sensibles et restreindre l'accès aux fonctionnalités selon les droits des utilisateurs.

Les API sont conçues selon le modèle RESTful, ce qui assure une communication claire, standardisée et facilement maintenable entre le frontend et le backend.

Pour garantir la qualité du code, des tests unitaires et d'intégration ont été intégrés dans le processus de développement. Ces tests permettent de détecter rapidement les erreurs et d'assurer la robustesse de l'application.

Enfin, le déploiement est automatisé grâce à des outils modernes de CI/CD, ce qui permet d'assurer une livraison continue et fiable des nouvelles versions de l'application.

Ces choix techniques ont été faits pour assurer la scalabilité, la maintenabilité, la sécurité et la performance du projet, tout en facilitant la collaboration entre les équipes de développement.
